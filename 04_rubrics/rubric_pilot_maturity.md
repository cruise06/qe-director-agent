# Rubric Pilot Maturity

## Maturity levels
- M0 Starting
- M1 Running
- M2 Stable
- M3 Repeatable

## M0 Starting
Use this when:
- Quality involvement is mostly reactive
- Release process is not yet stable
- Monitoring or metrics are incomplete
- Cross-team criteria are inconsistent

## M1 Running
Use this when:
- Basic pilot workflow is in place
- Some quality activities are repeatable
- Core monitoring exists
- Risks are visible but not yet consistently controlled

## M2 Stable
Use this when:
- Release readiness is mostly consistent
- Quality criteria are shared across teams
- Monitoring and pilot metrics support decisions
- Incident follow-up is mostly closed-loop

## M3 Repeatable
Use this when:
- Pilot workflow is standardized
- Quality gates and roles are clear
- Metrics, monitoring, and actions are systematic
- The pilot pattern can be reused by other teams

## Required output
- Maturity level
- Top issues
- Recommended actions
- Priority
- Director reminder
