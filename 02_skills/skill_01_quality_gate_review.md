# Skill 01 Quality Gate Review

## Purpose
Evaluate whether a release is ready to proceed from a quality engineering perspective.

## Input
- Project name
- Release name
- Planned release time
- Change scope
- Known risks
- Defect status
- Test coverage
- Regression result
- Monitoring readiness
- Rollback plan
- Current blockers
- Expected output

## Output
- Conclusion
- Key risks
- Analysis
- Recommended actions
- Priority
- Director reminder

## Decision options
- Go
- Conditional Go
- No Go

## Core review points
- Are there any unresolved blocker issues?
- Is regression coverage sufficient?
- Is rollback feasible?
- Is monitoring ready?
- Are key risks explicitly acknowledged?
