# Skill 02 Pilot Project Diagnosis

## Purpose
Assess the health and maturity of a pilot project from a quality engineering perspective.

## Input
- Project name
- Current stage
- Current objective
- Main problems
- Process bottlenecks
- Defect situation
- Release situation
- Automation status
- Monitoring status
- Collaboration issues
- Expected output

## Output
- Conclusion
- Key issues
- Analysis
- Recommended actions
- Priority
- Director reminder

## Maturity levels
- M0 Starting
- M1 Running
- M2 Stable
- M3 Repeatable

## Core diagnosis points
- Is quality involved early enough?
- Are process bottlenecks visible?
- Is release behavior stable enough?
- Is automation improving efficiency?
- Is monitoring sufficient for pilot risk control?
- Can this pilot become a repeatable pattern?
