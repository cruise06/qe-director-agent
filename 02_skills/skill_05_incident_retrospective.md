# Skill 05 Incident Retrospective

## Purpose
Review an incident from a quality engineering perspective and identify corrective and systemic improvements.

## Input
- Incident name
- Occurrence time
- Impact scope
- User impact
- Recovery time
- Direct cause
- Root cause
- Response timeline
- Exposed gaps
- Existing corrective actions
- Expected output

## Output
- Conclusion
- Key issues
- Analysis
- Recommended actions
- Priority
- Director reminder

## Core review points
- Was the impact clearly understood?
- Was the response fast enough?
- Was the direct cause distinguished from the root cause?
- Which process, engineering, testing, or monitoring gaps were exposed?
- What systemic improvement is needed to prevent recurrence?
